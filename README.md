# Compressed Sensing: Sparse Signal Recovery via Iterative Thresholding Methods
**Report (PDF):** [[Zenodo DOI]()]  
**Code Repository:** [[GitHub link](https://github.com/jacekschmidt/Compressed_Sensing/edit/main/README.md)]
## Overview

This project investigates compressed sensing as a framework for recovering sparse signals from underdetermined linear systems. In many practical settings, signals of interest are sparse or compressible when expressed in an appropriate basis, allowing accurate reconstruction from significantly fewer measurements than traditional sampling methods require.

The focus of this work is on iterative thresholding algorithms, particularly Iterative Hard Thresholding (IHT) and Normalised Iterative Hard Thresholding (NIHT). These methods provide computationally efficient approaches to sparse recovery and are well-suited to large-scale problems.

The project combines theoretical motivation with numerical experimentation, including phase transition analysis to evaluate recovery performance under varying sparsity and measurement regimes.

---

## Mathematical Model

We consider the problem of recovering a sparse vector x ∈ R^n from measurements:

A x = b

where A ∈ R^(m × n) with m < n.

This system is underdetermined, so infinitely many solutions exist. Compressed sensing imposes a sparsity constraint:

- Only k ≪ n entries of x are nonzero

The goal is to recover the sparsest solution consistent with the data.

---

## Algorithms Implemented

### Steepest Descent Least Squares (SDLS)

- Baseline iterative method
- Minimises ||Ax - b||²
- Does not enforce sparsity

### Iterative Hard Thresholding (IHT)

- Alternates between gradient descent and sparsity projection
- Keeps only the k largest coefficients at each step

### Normalised Iterative Hard Thresholding (NIHT)

- Variant of IHT with adaptive step size
- Improves convergence stability and recovery performance

---

## Key Components

### Thresholding Operator

- Retains k largest magnitude entries
- Sets all others to zero
- Enforces sparsity constraint directly

### Random Sensing Matrices

- Gaussian random matrices used
- Optional column normalisation

### Recovery Criterion

Recovery is considered successful if:

||x - x_recovered||² / ||x||² < tolerance

## Experiments

### Residual Landscapes

Contour and 3D visualisations of the residual:

||Ax - b||²

These illustrate the geometry of the optimisation problem and how iterative methods navigate it.

### Sparse Recovery Tests

Random sparse signals are generated and reconstructed using:

- IHT  
- NIHT  

Performance is evaluated based on reconstruction accuracy.

### Phase Transition Analysis

Phase transition diagrams are generated by varying:

- Number of measurements (m)  
- Sparsity level (k)  

Each point represents the probability of successful recovery over multiple trials.

---

## Results

### Phase Transition Behaviour

A sharp boundary is observed separating:

- High-probability recovery region  
- Failure region  

This demonstrates the fundamental limits of compressed sensing recovery.

### Effect of Column Normalisation

Normalising the columns of A:

- Improves recovery performance  
- Produces a clearer and more favourable phase transition boundary  

### IHT vs NIHT

- NIHT consistently outperforms standard IHT  
- More stable convergence behaviour  
- Higher probability of successful recovery across parameter ranges  

---

## Key Observations

- Sparse structure enables recovery from highly undersampled systems  
- Algorithm performance depends strongly on matrix properties  
- Column normalisation significantly improves stability  
- NIHT provides a clear improvement over standard IHT  
- Recovery exhibits a sharp phase transition rather than gradual degradation
